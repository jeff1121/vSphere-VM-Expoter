# 專案開發原則 (Project Constitution)

本文件定義 vSphere-VM-Expoter 專案的核心開發原則。我們採用 **MVP (最小可行性產品)** 策略，避免過度設計，專注於提供核心價值。

## 1. 程式碼品質 (Code Quality)

*   **保持簡單 (KISS)**：程式碼應以最簡單的方式解決問題。避免過早優化或引入不必要的抽象層。
*   **可讀性優先**：程式碼是寫給人看的。使用清晰的變數命名和適當的註解。
*   **模組化**：功能應適度模組化，但不要為了模組化而模組化。僅在需要重用時才進行拆分。
*   **一致的風格**：遵循語言標準的程式碼風格 (Linting)，確保團隊協作順暢。

## 2. 測試標準 (Testing Standards)

*   **關鍵路徑測試**：優先為核心業務邏輯和關鍵路徑編寫單元測試。
*   **實用主義**：不追求 100% 的測試覆蓋率，而是追求測試的有效性。測試應能捕捉真實的錯誤。
*   **自動化**：測試應易於執行，並整合到開發流程中。

## 3. 使用者體驗一致性 (User Experience Consistency)

*   **直觀操作**：CLI 或介面設計應符合直覺，減少使用者的學習成本。
*   **清晰的錯誤訊息**：當發生錯誤時，提供具體、可操作的錯誤訊息，而不是堆疊追蹤 (Stack Trace)。
*   **最小驚喜原則**：系統行為應符合使用者的預期。

## 4. 效能需求 (Performance Requirements)

*   **夠用就好**：效能優化應基於實際瓶頸，而非猜測。在 MVP 階段，功能正確性優於極致效能。
*   **資源友善**：避免不必要的記憶體佔用或 CPU 消耗，特別是在處理大量 VM 匯出時。
*   **回應性**：長時間運行的操作應提供進度回饋，避免使用者認為程式已當機。
